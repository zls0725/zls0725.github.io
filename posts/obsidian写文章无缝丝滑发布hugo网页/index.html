1. 理解 Obsidian 和 Hugo 的结合
#

  • Obsidian 是一个强大的 Markdown 笔记工具，支持通过 frontmatter（YAML 格式的元数据）定义文章属性，例如标题、日期、标签等。
  • Hugo 是一个静态网站生成器，它依赖于 Markdown 文件和 frontmatter 来生成网站内容。
  • 通过配置 Obsidian 的模板和插件，以及 Hugo 的内容目录结构，可以让 Obsidian 的笔记直接兼容 Hugo 的格式，从而实现无缝发布。

2. 设置 Obsidian 模板以包含 Hugo 所需的 frontmatter
#

为了让 Obsidian 笔记直接适配 Hugo，你需要在 Obsidian 中创建一个模板，自动为新文章添加 Hugo 所需的 frontmatter。以下是一个典型的 Hugo frontmatter 模板示例：

步骤：
#

  1. 在 Obsidian 中创建模板：

    • 在 Obsidian vault 中创建一个模板文件夹（例如 Templates）。

    • 新建一个名为 Blog Post.md 的模板文件，内容如下：

      ---
title: "{{title}}"
date: "{{date:YYYY-MM-DD}}T{{time:HH:mm:ss}}+08:00"
draft: true
description: ""
tags: []
categories: []
---
      • title：文章标题，{{title}} 可以通过 Templater 插件动态填充。
      • date：Hugo 要求的日期格式，+08:00 表示东八区（可根据你的时区调整）。
      • draft: true：默认设置为草稿，防止未完成文章直接发布。
      • description、tags、categories：根据你的 Hugo 主题需求添加。

  2. 启用 Templater 插件：

    • 在 Obsidian 中安装并启用 Templater 社区插件。
    • 配置 Templater，设置模板文件夹路径（例如 Templates）。
    • 设置新笔记自动应用模板：当你创建新笔记时，Templater 会自动插入上述 frontmatter。

  3. 创建新文章：

    • 在 Obsidian 中创建一个新笔记（例如通过快捷键 Ctrl+N）。
    • 使用 Templater 命令（Insert template）插入 Blog Post 模板。
    • 填写标题、描述、标签等信息，frontmatter 会自动生成。

3. 配置 Obsidian 和 Hugo 的目录结构
#

为了无缝发布，你需要确保 Obsidian 的笔记直接存储在 Hugo 的 content 文件夹中，或者通过脚本/插件同步到 Hugo 的内容目录。

方法一：直接在 Hugo 的 content 目录中编辑
#

  1. 将 Obsidian vault 设置为 Hugo 的 content 目录：

    • 将你的 Obsidian vault 设置为 Hugo 项目中的 content 文件夹（例如 /path/to/hugo-site/content）。
    • 或者，在 Obsidian vault 中创建一个 Blog 文件夹，通过符号链接（symlink）将其链接到 Hugo 的 content 目录： 
      ln -s /path/to/obsidian-vault/Blog /path/to/hugo-site/content
    • 这样，你在 Obsidian 的 Blog 文件夹中创建的笔记会直接出现在 Hugo 的 content 目录中。

  2. 注意事项：

    • 确保 .obsidian 文件夹被 Hugo 忽略（在 Hugo 的 config.toml 中添加 ignoreFiles = [".obsidian"]）。
    • 如果需要保持部分笔记私有，可以在 Obsidian 中设置特定文件夹（如 Private）不同步到 Hugo。

方法二：使用插件或脚本同步
#

如果不想直接在 Hugo 的 content 目录中编辑，可以使用插件或脚本将 Obsidian 笔记同步到 Hugo。

  1. 推荐插件：

    • Hugo Publish：此插件可以将 Obsidian 笔记直接转换为 Hugo 兼容的 Markdown 文件，并同步到 Hugo 的 content 目录。它会自动处理内部链接、图片路径和 frontmatter。
    • Sync to Hugo：支持选择特定笔记同步到 Hugo，转换 Obsidian 标签为 Hugo 标签，并清理不需要的元数据。
    • Obsidian Git：通过 Git 自动提交和推送笔记到 Hugo 仓库，适合搭配 GitHub Pages 或其他 CI/CD 部署。

  2. 脚本同步：

    • 如果不想依赖插件，可以编写一个简单的 shell 或 Python 脚本，将 Obsidian 笔记复制到 Hugo 的 content 目录。例如： 
      cp -r /path/to/obsidian-vault/Blog/*.md /path/to/hugo-site/content/posts/
    • 或者使用更复杂的脚本（如 obsidian-to-hugo），支持转换 Obsidian 的 Wiki 链接为 Hugo 的短代码链接。

4. 自动发布到 Hugo 网站
#

为了实现无缝发布，你可以设置自动化流程，让文章保存后自动生成 Hugo 网站并部署。

方法一：使用 Obsidian ShellCommands 插件
#

  1. 安装 Obsidian ShellCommands 插件。
  2. 配置一个 shell 命令来复制笔记并构建 Hugo 网站。例如： 
    cp -r /path/to/obsidian-vault/Blog/*.md /path/to/hugo-site/content/posts/
cd /path/to/hugo-site
hugo
git add .
git commit -m "Publish new post"
git push
  3. 为该命令设置一个快捷键（例如 Ctrl+0），每次保存文章后按快捷键即可自动发布。

方法二：使用 GitHub Actions 或其他 CI/CD
#

  1. 将 Hugo 网站托管在 GitHub 仓库。
  2. 配置 GitHub Actions 工作流，监控 content 目录的变更，自动运行 hugo 命令并部署到托管服务（如 GitHub Pages、Netlify、Cloudflare Pages）。 示例 GitHub Actions 配置文件（.github/workflows/deploy.yml）： 
    name: Deploy Hugo Site
on:
  push:
    branches:
      - main
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: '0.107.0'
      - name: Build
        run: hugo --minify
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public
  3. 在 Obsidian 中使用 Obsidian Git 插件，自动提交笔记到 GitHub 仓库，触发 CI/CD 部署。

方法三：使用 Cloudflare Pages 或 Netlify
#

  • 配置 Cloudflare Pages 或 Netlify 监控你的 Hugo 仓库。
  • 每次推送新文章到 GitHub 后，Cloudflare Pages 或 Netlify 会自动构建并部署 Hugo 网站。

5. 处理 Hugo 和 Obsidian 的兼容性问题
#

  • Wiki 链接：Obsidian 使用 [[Wiki链接]]，而 Hugo 需要标准 Markdown 链接或短代码。可以使用 obsidian-to-hugo 脚本或插件将 [[Wiki链接]] 转换为 Hugo 的 。
  • 图片路径：Obsidian 的图片路径可能与 Hugo 的 static 目录不兼容。确保图片存储在 Hugo 的 static 文件夹中，并使用相对路径。插件如 Hugo Publish 可以自动处理图片路径。
  • 嵌套目录：Hugo 需要 _index.md 文件来支持嵌套目录结构。如果你的 Obsidian vault 有深层目录，可以使用脚本自动生成 _index.md 文件。

6. 推荐工作流程
#

  1. 在 Obsidian 中使用 Templater 创建新文章，自动插入 Hugo 兼容的 frontmatter。
  2. 将文章保存在 Obsidian vault 的 Blog 文件夹中，该文件夹通过符号链接或插件同步到 Hugo 的 content 目录。
  3. 使用 Obsidian ShellCommands 或 Obsidian Git 插件，通过快捷键触发同步和发布。
  4. 配置 GitHub Actions 或 Cloudflare Pages 自动构建并部署 Hugo 网站。

7. 注意事项
#

  • 隐私保护：如果你的 Obsidian vault 包含私人笔记，确保只同步特定文件夹（如 Blog）到 Hugo。可以使用 frontmatter（如 publish: true）标记需要发布的笔记。
  • 插件兼容性：某些插件（如 ShellCommands）在 Flatpak 版本的 Obsidian 上可能有问题，建议使用 .deb 或 .appimage 版本。
  • 时区问题：确保 frontmatter 中的日期格式与 Hugo 要求一致（例如 YYYY-MM-DDTHH:mm:ss+08:00）。
  • 主题兼容性：不同 Hugo 主题对 frontmatter 的要求不同，检查你的主题文档，确保 Obsidian 模板匹配主题需求。

8. 推荐资源
#

  • Obsidian 插件：
    • Hugo Publish：一键发布到 Hugo。
    • Sync to Hugo：同步选定笔记到 Hugo。
    • Templater：自动插入 frontmatter。
    • Obsidian Git：自动提交到 Git 仓库。
  • 脚本和工具：
    • obsidian-to-hugo：转换 Obsidian 笔记为 Hugo 格式。
    • 自定义 Python/shell 脚本：处理嵌套目录、图片和链接。
  • 教程：
    • Jacob Kaplan-Moss 的博客：详细介绍 Obsidian 和 Hugo 的集成。

    • Nick Gracilla 的博客：分享 Templater 和 Hugo 的设置方法。
